/community direction notes

Engagement layer for the current Garden · 3 in-route tabs · same shell as Hub + Garden

What lives at /community

/community is the engagement layer for the current Garden — not a portfolio of many gardens. Same shell as Hub and Garden (light AppBar, floating NavBar pill, FAB). Same warm cream canvas with subtle pale-green wash. 3 tabs only: Activity · Proposals · People.

Cross-route flow: operator drafts in Garden › Treasury → publishes to Community › Proposals → community engages (votes, comments) → decided proposal lands in Community › Activity timeline.

TabContains
Activity Header strip with 3 chip-shaped stats (top contributors / proposals open / milestones this month) — replaces a separate Overview tab. Filter chips (All · Work · Proposals · Milestones · Discussions). Chronological feed with typed cards: Work (collage + author + category badge), Decided proposal (title + tally bar + decision badge), Milestone (compact, celebration accent — no banner), Discussion (thread + comments + last-replier). Stone-colored date dividers. Pagination: Load 20 more or month-jumper.
Proposals Filter chips (Active · Voting · Closed · All — default = Active + Voting). Search by title or proposal ID. Cards: title + 1-line summary + status chip + tally bar + time-remaining/closed-Xd-ago + creator + comment count. Operator affordances per status. Empty state per filter.
People Header strip with total + breakdown (gardeners · supporters · contributors). Filter chips. Person rows with engagement chips (work / supporter $ / votes / discussions) + last-active. Different lens from Garden › Members — engagement angle, not role-config.
