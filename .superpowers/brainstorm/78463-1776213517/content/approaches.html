Admin UI/UX Fix — Three Approaches

12 issues identified from audit. Pick the approach that matches your appetite for structural change.

A

Targeted Fixes (Minimal Change)

Fix each issue individually with smallest possible code changes. Widen RightSheet via CSS clamp tweak, add px-4 gutters to MainSheet, conditionally hide tabs with isDesktop, add tooltip wrappers to icons.

Pros

  • Smallest diff, lowest risk
  • Each fix is independently reviewable
  • Fast to implement (~2-3 hours)

Cons

  • RightSheet still serves dual purposes (profile + settings in one component)
  • Width logic stays hardcoded in inline styles
  • Doesn't address root cause of icon contrast issues
B

Sheet System Refactor (Recommended)

Split AccountSheet into ProfileSheet + SettingsSheet as separate RightSheet instances. Move sheet width to CSS custom properties on the canvas grid. Unify icon buttons into a shared TopBarIconButton component with tooltip, M3 state layer, and consistent sizing.

Pros

  • Clean separation: profile ≠ settings (your request)
  • Responsive width via CSS tokens — one source of truth
  • Icon button component eliminates inconsistency at the root
  • Scales well for future sheets (e.g., activity feed)

Cons

  • Medium diff — touches shared + admin packages
  • Need to handle sheet orchestrator for 3 content types
  • ~4-6 hours of work
C

Full Layout System Polish

Everything in B, plus: define a canvas layout token system (gutters, max-widths, sheet widths, bottom padding) as CSS custom properties. Refactor MainSheet to use these tokens. Add workspace-aware sheet styling. Full M3 audit of all sheet content panels.

Pros

  • Most systematic — all spatial values from one token set
  • Future layout changes are trivial (change one variable)
  • Sheet content gets proper M3 treatment

Cons

  • Largest scope — touches CSS foundation
  • Risk of scope creep into unrelated layout areas
  • ~6-10 hours
