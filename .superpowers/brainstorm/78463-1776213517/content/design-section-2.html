Design Section 2: TopBar Icon Buttons & Tooltips

Unified icon button component with M3 state layer, tooltips, and consistent sizing across all actions.

2a. TopBarIconButton Component

Replace the current ICON_BTN className string with a proper component that wraps each action icon.

TopBarIconButton — shared component in TopContextBar.tsx
// Every action icon — bell, gear, profile — uses this
<TopBarIconButton
  tooltip="Notifications"
  onClick={onOpenNotifications}
>
  <RiNotification3Line />
</TopBarIconButton>

2c. Styling Spec

M3 Icon Button Anatomy
Container: h-10 w-10 rounded-full
Color: text-text-strong (not text-sub)
Hover: bg-[rgb(var(--m3-on-surface)/0.08)]
Press: bg-[rgb(var(--m3-on-surface)/0.12)] scale-95
Hover scale: scale-105
Transition: 150ms spring-micro
Focus ring: ring-2 ring-ws-primary
Tooltip: Radix Tooltip, 400ms delay, bottom placement

Key change: The profile avatar is no longer a special case. Instead of a UserAvatar component with its own hover/layout, the profile action uses the same TopBarIconButton with a person icon (RiUserLine). The avatar with initials moves into the profile sheet content, not the top bar.
