Desktop: Clicking person icon opens ProfileSheet (title: "Profile"). Clicking gear opens SettingsSheet (title: "Settings"). Each is its own RightSheet with a title header — no tabs.
Mobile: Profile route renders full-page view with "Account" / "Settings" tabs via AdminTabRail. Tab labels change from "Profile"/"Settings" to "Account"/"Settings".

3b. Navigation Bar Loading Gate

Fix: Don't render nav until garden data is loaded
// In CanvasLayout, the nav render block becomes:
<div className="canvas-area-bottom">
  {eligibleGardensLoaded && visibleSlotCount > 0 && (
    <FabAwareNavigationBar ... />
  )}
</div>

Simple guard: nav only renders after eligibleGardensLoaded is true. No flash — the spinner/loading state handles the wait, then nav appears with correct slot count.

Two entry points for garden creation:
1. GardenChip dropdown — always includes "Create Garden" option, even when no gardens exist. The chip shows "No Garden Access" but is still tappable to open the dropdown.
2. Empty state CTA — prominent "Create Garden" button below the description. Always visible (removed the canCreateGarden gate on the button — any connected wallet should be able to start creation).

3d. Sheet Content Styling Fixes

With the wider sheet (320-480px), most content overflow issues resolve automatically. Remaining fixes:

  • Profile panel: Avatar centered, name/role below, wallet address in a copy-able card — all already exist but were clipped
  • Settings panel: Theme selector grid gets room to show 3 columns, network info readable
  • Notification panel: Empty state gets proper centering at the new width
  • All panels: Consistent p-5 padding, gap-5 section spacing
