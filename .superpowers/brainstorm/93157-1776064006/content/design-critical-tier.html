Architecture note: All admin overrides follow the same pattern — new components in packages/admin/src/components/ that accept the same data model as the shared primitives but render M3-strict visual anatomy. This means:
  • Zero changes to packages/shared/
  • Admin views swap imports (e.g. CanvasStageTabRail → AdminTabRail)
  • Client PWA remains completely unaffected
  • Liquid Glass surface treatment (blur, saturation) applied only to shell chrome, not to M3 component anatomy
