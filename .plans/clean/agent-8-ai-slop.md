# Agent 8 — AI Slop, Stubs, and Unhelpful Comments (DRY-RUN)

**Scope:** `packages/{client,admin,shared,agent,indexer,ops}/src` (excluding `__tests__`, `*.stories.tsx`, vendored `packages/contracts/lib/**`).

## 1. Executive Summary

The codebase is unusually clean for AI-slop patterns. No `debugger;`, no `generated by Claude/GPT/Copilot` markers, no stubs thrown in production paths (the handful of `not implemented` errors are deliberate xstate placeholder actors overridden by `.provide()`). Debug leftovers are near-zero — 2 `console.debug` calls violate the repo's Rule 12 in `AppSettings.tsx`; the remaining `console.*` hits in `packages/contracts/script`, `packages/ops/src/index.ts`, and `packages/agent` are legitimate CLI/bootstrap logging or docstring examples.

What *does* exist at scale is **JSDoc hypertrophy and WHAT-style narration of section flow**. ~4,141 single-line comments across prod source in `{client,admin,shared}/src`, and ~1,157 JSDoc blocks. 648 `@param`/`@returns`/`@example`/`@module` tags in `shared` alone. Most are benign, but several categories read like AI autocompletion habit rather than intent — particularly in `shared/src/utils/` helper modules.

Rough removable/shrinkable LOC estimate:
- WHAT-style narrator comments (`// Handle X`, `// Render Y`, `// Filter state`): **~290 lines**, heavy in `client/src/views/**` and `admin/src/views/**`.
- Dash-style section dividers *inside* a single function body (not barrel files): **~50 lines** in `admin/src/components/AdminTextField.tsx`, `AdminListItem.tsx`, `shared/src/components/Audio/*`.
- Redundant JSDoc `@param`/`@returns` on obvious getters/forwarders: **~150–200 lines** across `shared/src/utils/eas/explorers.ts`, `utils/blockchain/address.ts`, `utils/blockchain/chain-registry.ts`, `utils/action/parsers.ts`.
- Commented-out code blocks inside TODOs: **~12 lines** in `shared/src/modules/transactions/embedded-sender.ts`.
- Console.log/debug violations: **2 lines** in `client/src/views/Profile/AppSettings.tsx`.

**Rough deletable budget: 450–550 LOC** of comment/JSDoc noise without touching executable code. No dead stubs to remove. Executable code is very tight — no wrapper-forwarder pattern found, no `export *` barrel-spam in *prod* source (confined to `__mocks__/**` and a leaf-level pattern in `client/src/components/**/index.ts` which is reasonable for co-located feature folders).
