- **File**: `packages/shared/src/hooks/hypercerts/hypercert-contracts.ts:44-52`
- **Risk score**: 9.0 (impact=3 x likelihood=1 x staleness=3.0)
- **Confidence**: HIGH
- **Issue**: Chain-specific contract addresses hardcoded as fallback map (`HYPERCERT_MINTER_BY_CHAIN`). Silent staleness risk if Hypercerts protocol redeploys. No automated health check or registry comparison exists. This finding has been open since ~2026-02-18 (8 consecutive audits). Risk score exceeds 8.0 threshold.
- **Recommendation**: This finding has exhausted normal escalation. Either:
  (a) Mark as **ACCEPTED** -- the fallback is intentionally conservative and the Hypercerts protocol is stable.
  (b) Mark as **DEFERRED** with rationale ("Hypercerts has no on-chain registry; addresses are canonical").
  (c) Add a CI health check comparing these addresses against the Hypercerts deployment artifacts on GitHub.
  **Without a decision, this escalates to CRITICAL next audit.**

### H2. Agent typecheck broken by useStrategyRate type assertion (NEW)

- **File**: `packages/shared/src/hooks/vault/useStrategyRate.ts:40`
- **Risk score**: 6.0 (impact=3 x likelihood=3 x staleness=1.0)
- **Confidence**: HIGH
- **Issue**: `const tupleData = query.data as readonly unknown[]` fails TypeScript strict checking. The `AAVE_V3_POOL_ABI` defines `getReserveData` with named tuple components, so wagmi/viem infers `query.data` as `{ configuration: bigint; liquidityIndex: bigint; currentLiquidityRate: bigint; ... }` (an object), not a tuple array. Casting directly to `readonly unknown[]` is rejected by TS2352 because the types don't overlap. The `bun run --filter '@green-goods/agent' typecheck` command now fails with exit code 2.
- **Recommendation**: Access the named field directly instead of tuple indexing:
  ```typescript
  const data = query.data;
  const rawRate = data?.currentLiquidityRate;
  ```
  Or cast through `unknown` first: `(query.data as unknown as readonly unknown[])[2]` (less clean).
