### 2. React Hook Dependency Array Issues (HypercertWizard)

**Location**: `packages/admin/src/components/hypercerts/HypercertWizard.tsx:296-298, 445-448`
**Description**: Multiple `useEffect` and `useMemo` hooks have `wizardState.*` as dependencies which causes the linter to warn about outer scope values. This indicates potential stale closure issues.

**Recommendation**: Extract individual values before the hook instead of accessing from `wizardState` object inside:

```typescript
// Before (problematic)
useEffect(() => {...}, [wizardState.title, wizardState.description]);

// After (correct)
const { title, description } = wizardState;
useEffect(() => {...}, [title, description]);
```

### 3. aria-label Accessibility Issue

**Location**: `packages/client/src/components/Navigation/LandingHeader.tsx:39`
**Description**: Invalid ARIA property value pattern using template literal.

**Current**:
```tsx
aria-label={`Visit our ${title}`}
```

**Fix**: This is actually valid per spec. The linter warning is a false positive - string interpolation is allowed for aria-label. Consider adding an oxlint ignore comment.
