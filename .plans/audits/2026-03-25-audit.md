### H1. Hardcoded Hypercert Minter addresses (STILL OPEN, **CHRONIC** -- open 9 cycles)

- **File**: `packages/shared/src/hooks/hypercerts/hypercert-contracts.ts:44-52`
- **Risk score**: 9.0 (impact=3 x likelihood=1 x staleness=3.0)
- **Confidence**: HIGH
- **Issue**: Chain-specific contract addresses hardcoded as fallback map (`HYPERCERT_MINTER_BY_CHAIN`). Silent staleness risk if Hypercerts protocol redeploys. No automated health check or registry comparison exists. This finding has been open since ~2026-02-18 (9 consecutive audits). Risk score exceeds 8.0 threshold.
- **Recommendation**: This finding has exhausted normal escalation. Either:
  (a) Mark as **ACCEPTED** -- the fallback is intentionally conservative and the Hypercerts protocol is stable.
  (b) Mark as **DEFERRED** with rationale ("Hypercerts has no on-chain registry; addresses are canonical").
  (c) Add a CI health check comparing these addresses against the Hypercerts deployment artifacts on GitHub.
  **Without a decision, this escalates to CRITICAL next audit.**

### H2. i18n locale coverage test failing (NEW)

- **File**: `packages/shared/src/i18n/en.json` (6 new keys), `packages/shared/src/i18n/es.json`, `packages/shared/src/i18n/pt.json`
- **Risk score**: 6.0 (impact=3 x likelihood=3 x staleness=1.0)
- **Confidence**: HIGH
- **Issue**: 6 new i18n keys were added to `en.json` but not to `es.json` or `pt.json`, causing 3 test failures in `locale-coverage.test.ts`. The missing keys are:
  - `admin.actions.createAction`
  - `admin.actions.noDescription`
  - `admin.assessments.opensInNewTab`
  - `admin.gardens.openJoining`
  - `admin.gardens.operatorBadge`
  - `app.sidebar.signOut`
- **Impact**: `bun run test` fails (3 assertion failures in shared). All admin views use `defaultMessage` props, so the UI falls back to English correctly at runtime.
- **Recommendation**: Add the 6 missing keys to both `es.json` and `pt.json` with translated values.
