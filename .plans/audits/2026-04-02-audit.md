### H2. Conviction hooks missing imports -- 5 hooks crash at runtime (NEW)

- **Files**:
  - `packages/shared/src/hooks/conviction/useSetConvictionStrategies.ts:12,26,34`
  - `packages/shared/src/hooks/conviction/useAllocateHypercertSupport.ts:13,27,40`
  - `packages/shared/src/hooks/conviction/useRegisterHypercert.ts:12,25,33`
  - `packages/shared/src/hooks/conviction/useDeregisterHypercert.ts:12,25,33`

- **Risk score**: 12.0 (impact=4 x likelihood=3 x staleness=1.0)
- **Confidence**: HIGH
- **Issue**: All 5 hooks (excluding `useSetDecay` which is correct) import `useProgressiveInvalidation` and `INDEXER_LAG_SCHEDULE_MS` but then call `useDelayedInvalidation` and reference `INDEXER_LAG_FOLLOWUP_MS` -- neither of which is imported. This causes `ReferenceError: useDelayedInvalidation is not defined` at runtime when any conviction mutation is invoked. The bug also breaks agent typecheck (2 of 4 errors). 22 shared tests fail from this single bug.
- **Evidence**: `useSetDecay` (which correctly uses `useProgressiveInvalidation`) is the only hook that passes its tests.
- **Recommendation**: In each of the 5 files, change the import from:
  ```typescript
  import { INDEXER_LAG_SCHEDULE_MS, queryInvalidation } from "../query-keys";
  import { useProgressiveInvalidation } from "../utils/useTimeout";
  ```
  to:
  ```typescript
  import { INDEXER_LAG_FOLLOWUP_MS, queryInvalidation } from "../query-keys";
  import { useDelayedInvalidation } from "../utils/useTimeout";
  ```
