## High Findings

### H1. Hook defined outside `@green-goods/shared` [STILL OPEN - NEW]

- **File**: `packages/admin/src/components/Hypercerts/HypercertWizard/useWizardData.ts` (459 LOC, confirmed imported by `packages/admin/src/components/Hypercerts/HypercertWizard/index.tsx:16`)
- **Risk score**: Impact 3 x Likelihood 3 x Staleness 1.0 = **9.0**
- **Issue**: Violates the architectural invariant in `CLAUDE.md` ("ALL React hooks MUST live in `@green-goods/shared`") and the lint guard at `.claude/scripts/validate-hook-location.sh`. The file is a 459-line composite hook that orchestrates wizard state, drafts, minting, navigation blocking, and step navigation - exactly the kind of business logic the architecture wants centralised for reuse (client will eventually need the same flow).
- **Recommendation**: Move to `packages/shared/src/hooks/hypercerts/useWizardData.ts`, export via the shared barrel, adjust the import in `HypercertWizard/index.tsx`. The internal helpers (`useValidationMessage`, `useWizardSteps` in `wizardSteps.ts`) also live in admin and should move with it.

### H2. Canonical Admin components have zero consumers [STILL OPEN - NEW]

- **Files**:
  - `packages/admin/src/components/AdminBadge.tsx`
  - `packages/admin/src/components/AdminFab.tsx`
  - `packages/admin/src/components/AdminListItem.tsx`
  - `packages/admin/src/components/AdminTooltip.tsx`
- **Risk score**: Impact 3 x Likelihood 2 x Staleness 1.0 = **6.0** (High because the design system declares them canonical)
- **Issue**: The design skill (`.claude/skills/design/prompt-contract.md`, `ARCHITECTURE.md`, `defect-grammar.md`) lists these as 4 of the 13 canonical `Admin*` wrappers, yet `knip` finds no runtime imports - only their own `.stories.tsx` self-references. Either the design system is advertising primitives that do not exist in practice, or the consuming views never got migrated. Both interpretations are architectural drift.
- **Recommendation**: Decide per component:
  1. If a canonical component: identify the views that should use it and file a tracking issue to migrate them (`AdminFab` is referenced in `FabProvider` wiring per prompt-contract; `AdminTooltip`/`AdminListItem`/`AdminBadge` should back the defect-grammar mappings like "the tooltip", "the row", "the badge").
  2. If retired: remove the files and update `prompt-contract.md § Canonical Component Palette`, `ARCHITECTURE.md`, and `defect-grammar.md` in the same commit so the skill and the code stay in sync.
- **Ground truth**: `bun run check:design-tokens` is green, so the token spec has not drifted - but the component palette has.
