# QA Evidence

Future visual QA evidence for the Client PWA Design-System Transition belongs here.

Required future coverage:

- Stage 1 drawer and overlay screenshots/notes.
- Stage 2 PWA status map screenshots/notes.
- Stage 3 splash and loading motion screenshots/notes.
- Stage 4 media overlay and upload progress screenshots/notes.
- Stage 5 lower-risk controls, cards, and forms screenshots/notes.
- Final installed-phone PWA signoff.

Use the executable visual QA matrix in `../../eval.md` when deciding which route or Storybook story
to capture.

Each note should include date, actor, route or Storybook story, state/data setup, viewport/device,
what was verified, known limitations, and screenshot paths when screenshot artifacts are committed.
