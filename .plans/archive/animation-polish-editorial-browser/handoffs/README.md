# Handoffs

Create one handoff per execution lane when this plan is promoted.

Each handoff should include:

- Scope actually touched.
- Commands run.
- Browser evidence gathered.
- Remaining proof limits.

## TDD Proof

For behavior-changing UI work, record:

- RED result: command or browser proof showing the existing gap.
- GREEN result: command or browser proof showing the corrected behavior.
- Proof limits: animation/perception checks that still require human signoff.
