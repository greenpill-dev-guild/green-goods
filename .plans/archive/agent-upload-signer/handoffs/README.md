# Handoffs

Keep lane handoffs short and factual. Use one file per lane:

- `claude-ui.md`
- `codex-state-api.md`
- `codex-contracts.md`
- `claude-qa-pass-1.md`
- `codex-qa-pass-2.md`

Each handoff should capture:

1. What changed
2. What remains
3. Validation run
4. Known risks or blockers
5. Repo-truth references from the active hub or reports, not tool-local memory claims
