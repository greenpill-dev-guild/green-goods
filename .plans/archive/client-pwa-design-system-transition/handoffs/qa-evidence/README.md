# QA Evidence

Visual QA evidence for the Client PWA Design-System Transition belongs here.

Required coverage:

- Stage 0 plan readiness gate note (planning artifacts only; no source edits).
- Stage 1 color foundation screenshots/notes plus the cross-shell public-browser smoke for `/`
  and `/gardens`.
- Stage 2 drawer and overlay screenshots/notes.
- Stage 3 loading and media motion screenshots/notes.
- Stage 4 lower-risk controls, cards, and forms screenshots/notes.
- Final installed-phone PWA signoff, or an explicit human-waiver/manual-device proof note.

Use the executable visual QA matrix in `../../eval.md` when deciding which route or Storybook story
to capture.

Each note should include date, actor, route or Storybook story, state/data setup, viewport/device,
what was verified, known limitations, and screenshot paths when screenshot artifacts are committed.
