# Internationalization

> Sub-file of the [ui skill](./SKILL.md).

---

## Architecture

Green Goods uses **react-intl** for UI string translation with 3 bundled locales (en, es, pt).

### Key Files

```
packages/shared/src/
  i18n/
    en.json    # English messages
    es.json    # Spanish messages
    pt.json    # Portuguese messages
  providers/
    App.tsx    # IntlProvider setup
```

## react-intl Usage

### Basic Patterns

```typescript
import { useIntl, FormattedMessage } from "react-intl";

// Component-based (in JSX)
<FormattedMessage id="work.submit" defaultMessage="Submit Work" />

// Imperative (for attributes, toast, etc.)
const intl = useIntl();
intl.formatMessage({ id: "work.title.placeholder", defaultMessage: "Enter work title" });
```

### ICU Message Syntax

```typescript
// Interpolation
<FormattedMessage id="garden.members" defaultMessage="{count} members" values={{ count: 5 }} />

// Pluralization
<FormattedMessage
  id="work.pending"
  defaultMessage="{count, plural, one {# pending work} other {# pending works}}"
  values={{ count }}
/>
```

### Adding New Messages

1. Add message ID + default text to `packages/shared/src/i18n/en.json`
2. Add translations to `es.json` and `pt.json`
3. Use in components via `FormattedMessage` or `useIntl`

```json
// en.json
{ "garden.join.button": "Join Garden" }
// es.json
{ "garden.join.button": "Unirse al Jardin" }
// pt.json
{ "garden.join.button": "Entrar no Jardim" }
```

## Number and Date Formatting

Use `Intl` APIs for locale-aware formatting — never format manually:

```typescript
new Intl.NumberFormat(locale).format(1234.56);           // "1,234.56" (en-US)
new Intl.DateTimeFormat(locale, { year: "numeric", month: "long", day: "numeric" }).format(date);
```

## Anti-Patterns

- **Never hardcode English-only strings** — use `FormattedMessage` or `useIntl`
- **Never format numbers/dates manually** — use `Intl` APIs
- **Never forget all 3 locale files** — en.json, es.json, pt.json must stay in sync
