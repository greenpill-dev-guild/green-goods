## Overview

Evaluators are the trust layer of Green Goods. Your role is to verify that the impact claims made by gardens and gardeners are backed by real evidence. You query on-chain data, follow attestation chains from work submissions to approvals, and produce analysis reports that funders and other stakeholders can rely on.

{/* IMAGE PLACEHOLDER */}

**How impact data flows:**

1. **Gardeners submit work** — photos, descriptions, and context about regenerative actions.
2. **Operators review and approve** — verified work becomes part of the garden's impact record via EAS attestations.
3. **You analyze the data** — query across gardens, verify evidence chains, and produce reports.

## Best Practices

- Always verify attestation chains end-to-end: work UID exists, approval refUID matches, attester is an authorized operator
- Cross-reference indexer data with EAS explorer to catch any indexing delays or inconsistencies
- Document your evaluation methodology so results are reproducible by other evaluators
- Start with a small sample set to validate your queries before running analysis at scale
- When evaluating across multiple chains, confirm you are using the correct EAS endpoint for each chain
