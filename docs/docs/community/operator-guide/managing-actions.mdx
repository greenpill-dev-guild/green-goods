# Managing Actions

<StatusBadge status="Live" />

## Overview

Actions live in the shared Action Registry, and gardens expose them based on domain fit and timing. In the current admin app, **creating and editing action definitions is a deployer-level flow** under `/actions`. Operators still need to understand actions because domain choice, instructions, timing, and acceptance criteria directly shape what gardeners can submit.

## Operator standards

- Use explicit acceptance criteria in instructions so review decisions stay consistent
- Keep action windows realistic for field conditions, connectivity, and seasonality
- Prefer short action titles and specific instructions with examples of acceptable evidence
- Use garden domains intentionally, because they are the gate that makes actions visible in a garden
- If the registry action is wrong and you do not have deployer access, escalate early instead of asking gardeners to guess
