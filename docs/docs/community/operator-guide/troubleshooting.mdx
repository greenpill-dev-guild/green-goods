# Operator Troubleshooting

<StatusBadge status="Live" />

## Common failure classes

<FeatureState
  title="Permission denied"
  status="Live"
  summary="Role mismatch or missing hat assignment blocks create/review operations."
>
  Confirm wallet/account role on the active chain and garden.
</FeatureState>

<FeatureState
  title="Route exists but action fails"
  status="Implemented (activation pending deployment)"
  summary="Several v1 flows are wired in UI but blocked by zero-address module deployments."
>
  Check `packages/contracts/deployments/*-latest.json` and `packages/indexer/config.yaml` before treating as runtime bug.
</FeatureState>

<FeatureState
  title="Indexer mismatch"
  status="Implemented (activation pending deployment)"
  summary="New module events will not appear until indexer contract addresses are non-zero and deployed."
>
  Compare deployed addresses and indexer network config, then redeploy indexer if needed.
</FeatureState>

## Escalation packet

Share these in incident reports:

1. Chain ID and garden address.
2. Route and exact action attempted.
3. Transaction hash (if any).
4. Console/network error snippet.
5. Deployment/indexer address check results.
