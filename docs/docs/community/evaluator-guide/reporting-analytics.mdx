# Reporting & Analytics

<StatusBadge status="Live" />

## Overview

Evaluators use reporting tools to transform raw attestation data into structured impact reports. Green Goods integrates with [Karma GAP](https://gap.karmahq.xyz/) to automate much of this process — approved work attestations feed directly into accountability-compatible report formats.

{/* IMAGE PLACEHOLDER: Reporting dashboard showing attestation metrics and export options (800x400) */}

This page covers the reporting surfaces available to evaluators, the monthly reporting loop, and how to map outputs to external frameworks like SDG indicators or funder-specific schemas.

### Data Sources

Evaluators can pull data from multiple sources:

- **Indexer GraphQL API** — Query gardens, work submissions, approvals, and membership data
- **EAS Explorer** — Verify individual attestation UIDs and their on-chain status
- **Admin Dashboard** — View garden-level analytics, approval rates, and work volume

- Always include attestation UIDs in reports so reviewers can independently verify claims
- Document any exclusions (rejected work, incomplete submissions) transparently
- Use consistent date ranges aligned with funder reporting cycles
- Archive exported data for longitudinal analysis across reporting periods
