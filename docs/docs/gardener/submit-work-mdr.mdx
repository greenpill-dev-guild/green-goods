Use this guide when you need to submit work from mobile or field conditions, including offline-first submission flows. This page also covers how drafts persist, how the submission queue works, and what to do when connectivity is limited.

## What good submissions include

- Clear before/after evidence where relevant.
- Direct alignment with action instructions.
- Accurate metrics and notes (no placeholders).
- Consistent location/timing details.

### Submission outcomes

- **Online + ready:** Submission is sent immediately.
- **Offline or network issue:** Submission is safely queued for retry.
- **Validation failure:** You get actionable error feedback before submit.

### Field tips

1. Save progress before leaving field conditions — drafts persist in your browser's local storage.
2. Reopen drafts and verify media is still attached before submitting.
3. Check queue/upload states before repeating submissions — duplicate submissions waste operator review time.
4. If a job fails repeatedly, retry from your work dashboard with corrected data.
