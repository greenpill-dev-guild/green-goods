## Fast triage

<FeatureState
  title="Validation errors"
  status="Live"
  summary="Missing required fields, invalid values, or oversized media prevent submit until fixed."
>
  Fix by reopening the draft, correcting highlighted fields, and resubmitting.
</FeatureState>

<FeatureState
  title="Network or offline failures"
  status="Live"
  summary="Submit fails when connectivity drops. The queue retries automatically when online."
>
  Confirm connection, then reopen the queue state and trigger retry if needed.
</FeatureState>

<FeatureState
  title="Signature or session issues"
  status="Live"
  summary="Passkey/wallet session may expire before mutation completes."
>
  Re-authenticate, then submit the same draft. Do not duplicate a new entry unless the old one is missing.
</FeatureState>

## Error-to-fix map

| Error Pattern | Likely Cause | Recommended Fix |
| --- | --- | --- |
| `media required` or image missing | Media not attached or local blob cleared | Reattach media and confirm preview before submit |
| `failed to submit work` | Network/API failure | Keep draft, reconnect, retry from queue |
| `unauthorized` or `forbidden` | Role/membership mismatch | Confirm you are in the target garden and action is active |
| `transaction rejected` | User rejected signature | Re-open submit flow and approve signature |

## Escalate with context

If retries keep failing, share these items with operators/developers:

1. Garden address and action UID.
2. Approximate timestamp.
3. Error text shown in UI.
4. Whether the submission is in `Queued`, `Failed`, or `Pending`.
