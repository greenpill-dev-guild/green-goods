Common questions about Green Goods, organized by role.

## General

<QuickAnswer question="What is Green Goods?">

Green Goods is an offline-first web app for documenting regenerative work, reviewing submissions, and producing verifiable on-chain records.

</QuickAnswer>

<QuickAnswer question="What chains are currently in scope?">

- Arbitrum One (`42161`) for production workflows. <StatusBadge status="Live" />
- Celo (`42220`) for production workflows. <StatusBadge status="Live" />
- Sepolia (`11155111`) for testnet and validation workflows. <StatusBadge status="Live" />

</QuickAnswer>

<QuickAnswer question="Can Green Goods work offline?">

Yes. Gardener workflows support offline drafts and queued sync when connectivity returns.

</QuickAnswer>

<QuickAnswer question="Is Green Goods free to use?">

- Gardener flows are passkey-first and gasless for common submission workflows.
- Operator actions that submit transactions can require network gas.

</QuickAnswer>

<QuickAnswer question="Is uploaded data private?">

No by default. Garden and attestation data is designed to be publicly verifiable. Do not include sensitive personal information in uploads.

</QuickAnswer>

## Gardener

<QuickAnswer question="How do I sign up?">

1. Open [greengoods.app](https://greengoods.app).
2. Choose passkey sign in.
3. Complete device biometric confirmation.

</QuickAnswer>

<QuickAnswer question="Do I need crypto to submit work?">

No. Gardener submission flows are designed to work without wallet management.

</QuickAnswer>

<QuickAnswer question="What if my submission is rejected?">

Use operator feedback to update evidence and resubmit. See:

- [Submit Work with MDR](../community/gardener-guide/uploading-your-work)
- [Common Errors](../community/gardener-guide/common-errors)

</QuickAnswer>

<QuickAnswer question="How do I track approval status and attestations?">

See [Track Status and Attestations](../community/gardener-guide/track-status-and-attestations).

</QuickAnswer>

## Troubleshooting

<QuickAnswer question="App does not load">

- Check connection.
- Refresh and clear browser cache.
- Confirm service availability in GitHub issues.

</QuickAnswer>

<QuickAnswer question="Media upload fails">

- Check file size and permissions.
- Retry with stable connection.
- Use offline draft flow if connectivity is unstable.

</QuickAnswer>

<QuickAnswer question="Transaction fails">

- Confirm correct network.
- Confirm wallet balance for gas.
- Retry with updated gas estimate.

</QuickAnswer>
