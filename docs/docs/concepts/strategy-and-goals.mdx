# Strategy & Goals

Green Goods pursues three strategic goals, each with distinct implications for development priorities, success metrics, and partnership strategy.

## Strategic Goals

### 1. Capital Formation (High Sustainability)

**Goal:** Create self-sustaining funding mechanisms that connect verified impact to capital flows.

**Mechanisms:**
- **Octant Vaults** — Yield-bearing deposits where returns fund conservation operations
- **Hypercerts** — Tokenized impact certificates that funders can purchase
- **RevNets** — Revenue-sharing token models for long-term community funding

**Implications for development:**
- Vault UX must be approachable for non-crypto funders
- Hypercert minting must be seamless for operators (not a blockchain ceremony)
- Impact data quality directly affects funder confidence and capital inflow

**Success metrics:**
- Total Value Locked (TVL) in vaults
- Hypercerts minted and listed per month
- Funder retention rate

### 2. Impact Reporting Accessibility (High Reach)

**Goal:** Make impact capture so frictionless that any field worker can participate, regardless of technical literacy or connectivity.

**Mechanisms:**
- **AI agents** on WhatsApp/SMS for communities without smartphones
- **Offline-first PWA** for low-connectivity environments
- **Passkey authentication** eliminating wallet/seed phrase barriers
- **Frictionless Hypercert minting** that abstracts blockchain complexity

**Implications for development:**
- Gardener-facing flows must work in under 2 minutes on 2G/3G
- Zero blockchain vocabulary in gardener-facing UI
- Agent-based reporting channels are the primary growth vector

**Success metrics:**
- Active gardeners per month
- Actions submitted per gardener
- Time-to-submit (target: under 2 minutes)

### 3. Onchain Governance & Reputation (High Coordination)

**Goal:** Build transparent, community-driven governance using on-chain reputation and signaling.

**Mechanisms:**
- **Unlock Protocol** — Token-gated access and credential badges
- **Hats Protocol** — Role-based access control (Operator, Gardener, Evaluator roles)
- **Gardens V2** — Conviction voting for community resource allocation
- **Conviction Voting** — Stake-weighted signaling for prioritizing actions

**Implications for development:**
- Role management must be intuitive for non-technical operators
- Governance features should enhance (not block) daily operations
- Reputation builds from verified actions, not token holdings

**Success metrics:**
- Governance participation rate
- Signal pool utilization
- Role transitions (Gardener → Operator progression)

## Economic Model

### Revenue Structure

| Source | Range | Notes |
|--------|-------|-------|
| Operator subscriptions | $49 – $1,000/mo per Garden | Scales with garden size and features |
| Funder dashboards | $2k – $50k/yr | Impact reporting and analytics |
| Yield from vaults | Variable | Octant vault returns fund operations |

### Cost Structure

| Cost | Monthly Estimate |
|------|-----------------|
| Gas (L2 transactions) | ~$30 |
| IPFS storage (Storacha) | ~$15 |
| Infrastructure | ~$100 |
| Monitoring/analytics | ~$50 |
| **Total operational cost** | **~$195/mo** |

### Break-Even Analysis

- **Minimum viable TVL:** ~$60k (covers operational costs from yield)
- **Target TVL:** $100k – $200k (self-sustaining operations + growth)
- **B2B2C model:** Operators onboard gardeners at effectively zero acquisition cost (passkeys + gas sponsorship)

## Competitive Moats

| Moat | Description |
|------|-------------|
| **Network effects** | More gardeners → more data → more funder confidence → more capital → more gardeners |
| **Switching costs** | Attestation history and reputation are on-chain and portable, but workflow integration is sticky |
| **Data moat** | Longitudinal impact data across 20+ communities builds unique evidence base |
| **Capital moat** | Yield-backed vaults create compounding returns that new entrants can't replicate immediately |

- **Octant** — Yield-bearing vault infrastructure

## Feature Scope

### In Scope (v1 Release)

- Offline-first work submission with photo evidence
- Operator review and approval workflows
- Hypercert minting from approved work batches
- Vault deposits with yield tracking
- Role management via Hats Protocol
- EAS attestation chains (Work → Approval → Assessment)
- Indexer for core state queries
- Admin dashboard for garden operations

### Out of Scope

- Carbon credit issuance or offset marketplace
- Self-custodial wallet flows for gardeners
- Real-time dashboards (acceptable lag is fine)
- Always-online features
- Direct fiat on/off-ramp integration

### Ship Later

- Agent-based impact reporting (WhatsApp/SMS bots)
- IoT + AI integrations for automated evidence capture
- RevNet tokenization for community revenue sharing
- Community stablecoins
- Bio-regional quadratic funding streams
- Unlock Protocol badge integration

## Active Grants & Programs

Green Goods development is supported through a combination of ecosystem grants and hackathon participation. Grant applications and proposals are managed through the Greenpill Dev Guild's internal processes and are not covered in public documentation.
