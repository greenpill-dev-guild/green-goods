# GreenWill

<StatusBadge status="Implemented (activation pending deployment)" />

## Overview
GreenWill badge support is present in registry contracts, shared data/hooks, admin action views, tests, and indexer event definitions. Current chain configs still use zero GreenWill addresses, so public availability should be described as deployment-gated. The separate `GreenGoodsBadge` EAS schema is a later portability layer, not a requirement for the initial three-badge GreenWill launch.

## Current surfaces

- `GreenWill.sol` models badge definitions and grants.
- Shared hooks read badge definitions, account badges, recent grants, and claim flows.
- The admin Actions surface includes a GreenWill panel.
- The indexer config includes GreenWill events, but configured network addresses are currently zero.

## Builder notes

- Do not present GreenWill badges as live on a chain until deployment artifacts and indexer config have non-zero addresses.
- For the initial launch, deploy only Genesis, First Work, and First Support through GreenWill + Unlock; keep the `GreenGoodsBadge` EAS schema in backlog/future portability work.
- Keep proposal/spec pages unlisted unless they are promoted into source-backed implementation docs.
- Make badge claims chain-aware in docs and UI.
