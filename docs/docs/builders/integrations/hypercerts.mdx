# Hypercerts

<StatusBadge status="Implemented (activation pending indexing)" />

## Overview
Hypercert support exists across deployed contract modules, marketplace adapter code, shared mint/listing hooks, metadata helpers, and admin wizard surfaces. Arbitrum and Sepolia artifacts have non-zero module and marketplace adapter addresses; Celo does not. Marketplace adapter events are not currently configured in the Green Goods indexer, so listing/trade claims should stay indexing-aware.

## Current surfaces

- Admin Hypercert wizard and preview components.
- Shared hooks for minting, allowlists, drafts, listings, approvals, trade history, and marketplace services.
- Shared metadata, validation, distribution, and Merkle helpers.
- Indexer events for Hypercert transfers and stored claims.

## Builder notes

- Keep certificate copy aligned with operator guide language.
- Use shared Hypercert hooks for data and transaction workflows.
- Treat marketplace-derived support badges and listings as chain- and indexer-aware; do not imply universal marketplace support.
