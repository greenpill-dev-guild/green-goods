## Overview
Karma GAP support exists in deployed Arbitrum/Sepolia contract modules, libraries, interfaces, and shared error handling. Celo artifacts keep the module at the zero address. Dedicated Karma GAP module events are not currently configured in the indexer, so reporting claims should remain indexing- and UI-gated.

## Builder notes

- Treat Karma GAP as non-blocking around core garden/work flows.
- Keep GAP errors mapped through shared contract-error utilities.
- Do not claim automatic reporting is live unless a deployed chain and UI path are verified.
