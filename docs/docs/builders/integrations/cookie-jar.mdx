# Cookie Jar Allocations

<StatusBadge status="Implemented (activation pending deployment)" />

## Overview
Cookie Jar support is implemented across contracts, shared hooks, and admin Hub components, but production availability depends on deployed module addresses and the connected garden configuration. Treat the code surface as real and the live product surface as activation-gated.

## Current surfaces

- `CookieJar.sol` owns the contract-side module boundary.
- Shared hooks cover accessible jars, garden jars, user jars, deposits, withdrawals, and admin operations.
- Admin Hub components expose deposit, withdraw, manage, and payout panels.
- Tests exist for the shared hooks and admin modal/payout surfaces.

## Builder notes

- Gate UI claims on non-zero module addresses and garden eligibility.
- Keep deposit/withdraw behavior routed through shared hooks rather than route-local contract calls.
- Align user-facing status with the community Cookie Jar guide when activation state changes.
