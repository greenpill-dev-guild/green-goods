# Docs Writing Guide

Use this guide when creating or editing canonical pages under `/gardener`, `/operator`, `/evaluator`, and `/developers`.

## UX writing rules

1. Keep one primary task per page.
2. Start with the first user action above the fold.
3. Keep sections short and scannable.
4. Use action verbs in headings.
5. End with one clear next action.

## Repetition reduction rules

1. Prefer reusable components over copy-pasted prose blocks.
2. Move shared endpoints and statuses into canonical reference pages.
3. Link to canonical sources instead of repeating long explanations.
4. If two sections repeat intent, collapse into one decision block.

## Hallucination resistance rules

1. Every non-reference page must include `source_of_truth` with real file paths.
2. Status labels must use the approved set only.
3. Distinguish implemented code from deployed/indexed reality.
4. Do not hardcode endpoints outside approved reference data/components.
